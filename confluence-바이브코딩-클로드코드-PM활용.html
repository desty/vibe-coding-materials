1. 세상이 바뀌었다 — 왜 지금 바이브코딩인가

2025년 초, AI 코딩 도구의 발전이 임계점을 넘었습니다. 더 이상 "코드를 조금 빠르게 쓰는" 수준이 아니라, 비개발자도 자연어만으로 동작하는 소프트웨어를 만들 수 있는 시대가 시작되었습니다.

이 변화는 개발자만의 이야기가 아닙니다. PM, 디자이너, 마케터 등 비개발 직군이 직접 프로토타입을 만들고, 데이터를 분석하고, 문서를 자동화하는 새로운 업무 방식의 문이 열렸습니다.

핵심 메시지 바이브코딩은 "개발자 되기"가 아닙니다. 내 업무를 AI와 함께 더 빠르고 정확하게 처리하는 새로운 방법입니다.

2. 바이브코딩, 30초 만에 이해하기

2025년 2월 Andrej Karpathy(전 Tesla AI Director, OpenAI 공동창업자)가 이런 트윗을 합니다:

"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

핵심은 간단합니다:

예전 방식

내가 원하는 걸 코드 문법으로 변환해서 한 줄씩 작성

수개월 학습 필요 / 문법 오류와 씨름

바이브코딩

내가 원하는 걸 자연어(한국어)로 설명하면 AI가 코드 생성

바로 시작 가능 / 결과 확인하고 피드백만

비유하자면, 바이브코딩은 "통역사와 함께 외국 식당에서 주문하는 것"과 같습니다. 외국어를 몰라도, 원하는 메뉴를 한국어로 설명하면 통역사가 알아서 주문해줍니다. 단, 나온 음식이 맞는지 확인하는 건 내 역할입니다.

PM 관점에서의 패러다임 변화

구분기존바이브코딩 이후
프로토타입개발팀에 요청 → 2~3주 대기직접 설명 → 30분 안에 확인
데이터 분석SQL 모르면 데이터팀에 의뢰"이 CSV에서 Top 10 뽑아줘"
문서 작성빈 문서에서 밑바닥부터"PRD 초안 만들어줘" → 수정
경쟁사 분석일일이 수동 정리"비교 매트릭스 만들어줘"
스프린트 리뷰Jira 수동 취합자동으로 티켓 정리 & 리포트

3. "ChatGPT 쓰고 있는데, 뭐가 다른 건데?"

여러분 대부분이 ChatGPT를 이미 쓰고 계실 겁니다. "회의록 요약해줘", "이메일 초안 써줘" 정도는 익숙하실 거예요. 그런데 Claude Code는 근본적으로 다른 물건입니다.

가장 큰 차이: "대화"가 아니라 "실행" ChatGPT는 텍스트를 주고받는 채팅입니다. Claude Code는 내 컴퓨터에서 직접 일을 하는 에이전트입니다. 파일을 만들고, 수정하고, 테스트하고, 에러를 고치고, Git에 커밋까지 합니다. 답변을 복사-붙여넣기 할 필요가 없습니다.

ChatGPT vs Claude Code: 결정적 차이 5가지

구분ChatGPT (지금 쓰는 것)Claude Code (오늘 소개하는 것)
동작 방식 질문하면 텍스트로 답변 지시하면 내 컴퓨터에서 직접 실행
파일 접근 첨부한 파일 1~2개만 프로젝트 폴더 전체를 읽고 수정
결과물 텍스트를 복사해서 써야 함 파일이 바로 생성/수정됨
외부 도구 연동 불가 (플러그인 제한적) Jira, Slack, GitHub, Drive에 직접 접속 (MCP)
자율성 한 번에 하나의 응답 스스로 계획 → 실행 → 에러 수정 → 완료까지 알아서

구체적으로 어떤 차이인지 봅시다

ChatGPT로 대시보드 만들기

1. "대시보드 코드 만들어줘" → 코드 텍스트 받음

2. 코드를 복사해서 파일에 붙여넣기

3. 에러 발생 → 에러 메시지를 다시 복붙해서 물어봄

4. 수정된 코드 다시 복붙...

5. 이 과정을 5~10번 반복

소요: 1~2시간, 좌절감 보너스

Claude Code로 대시보드 만들기

1. "대시보드 만들어줘" → 파일 자동 생성

2. 브라우저에서 바로 확인

3. "차트 색 바꿔줘" → 자동 수정

4. 에러가 나면 알아서 감지하고 수정

 

소요: 15~30분, 복붙 0회

도구 비교 (비유)

도구비유하는 일
ChatGPT아는 것 많은 친구"이렇게 하면 돼"라고 알려줌. 실행은 내가.
GitHub Copilot속삭이는 비서코드 에디터에서 다음 줄 제안
Cursor옆자리 주니어에디터 안에서 코드 수정 도와줌
Claude Code위임받은 시니어"맡겨주세요" → 알아서 하고 결과 보고

어디서 쓸 수 있나?

터미널, VS Code, 데스크톱 앱, 웹 브라우저(claude.ai/code), 심지어 Slack에서도 사용할 수 있습니다. ChatGPT처럼 웹에서 바로 시작할 수도 있고, 터미널에서 프로 모드로 쓸 수도 있습니다.

4. 실제 사례: "이게 된다고?"

사례 1

비개발자가 30분 만에 SaaS 랜딩페이지 완성

마케팅 담당자가 Claude Code에게 "우리 서비스 랜딩페이지를 만들어줘. Hero 섹션, 기능 소개 3개, 가격표, CTA 버튼 포함. 모던하고 깔끔하게"라고 요청. 30분 만에 반응형 웹페이지가 완성되었고, "버튼 색깔 파란색으로 바꿔줘", "가격을 달러에서 원화로 바꿔줘" 같은 피드백으로 수정 완료.

사례 2

PM이 유저 인터뷰 10건을 5분 만에 인사이트 추출

유저 인터뷰 트랜스크립트 10건을 Claude에 넘기고 "공통 Pain Point 찾아줘. 빈도순으로 정리하고, 각 Pain Point별 대표 인용문도 뽑아줘"라고 요청. 5분 만에 Pain Point 7개, 빈도 분석, 직접 인용문이 포함된 리포트가 나옴.

사례 3

1인 창업자가 MVP를 혼자 만들어서 런칭

개발 경험 없는 창업자가 Claude Code로 할일 관리 앱 MVP를 3일 만에 완성. 프론트엔드(React), 백엔드(Supabase), 배포(Vercel)까지 전부 자연어 지시만으로 진행. Product Hunt에서 상위 10위 진입.

사례 4

스프린트 리뷰를 자동으로 정리

PM이 Jira MCP를 연동하고 "이번 스프린트 완료된 티켓을 카테고리별로 정리하고, 릴리스 노트 초안 만들어줘"라고 요청. 2분 만에 기능 추가/버그 수정/개선으로 분류된 릴리스 노트가 완성. 이전에는 매주 1시간씩 걸리던 작업.

사례 5

CSV 데이터를 인터랙티브 대시보드로 변환

사업팀에서 받은 매출 CSV를 Claude에 넘기고 "월별 매출 추이, 제품군별 비중, 전년 대비 성장률을 대시보드로 만들어줘"라고 요청. 10분 만에 차트가 포함된 HTML 대시보드가 완성. 경영진 보고에 바로 사용.

5. PM이 바로 쓸 수 있는 6가지 시나리오

시나리오 1: PRD 작성 도우미

Claude는 단순히 문서를 채워주는 게 아닙니다. 소크라틱 방식으로 여러분이 놓친 부분을 질문하면서 함께 PRD를 완성합니다.

나 (PM)
우리 앱에 소셜 로그인 기능을 추가하려고 해. PRD를 같이 만들어보자.
Claude
좋습니다! 몇 가지 먼저 확인하고 싶은 게 있어요.
1. 타겟 사용자는 누구인가요? (B2B/B2C)
2. 어떤 소셜 로그인을 지원하고 싶으세요? (Google, Apple, Kakao 등)
3. 기존 이메일 계정과 소셜 계정 연동이 필요한가요?
4. 보안 요구사항 레벨이 어떻게 되나요?
나
B2C, Google이랑 카카오, 기존 계정 연동 필요, 보안은 표준 수준
Claude
이해했습니다. 추가로 Edge case도 고려해볼게요. 같은 이메일로 Google과 카카오를 모두 연동했을 때 처리 방식, 소셜 로그인 서비스 장애 시 fallback 등도 포함하겠습니다. PRD 초안을 작성할게요...

시나리오 2: 프로토타입 빠르게 만들기

디자이너에게 요청하기 전, 또는 개발 킥오프 미팅 전에 아이디어를 시각적으로 보여줄 프로토타입을 직접 만들 수 있습니다.

나
대시보드 프로토타입 하나 만들어줘. 왼쪽에 네비게이션, 메인에 KPI 카드 4개(MAU, MRR, Churn Rate, NPS), 아래에 주간 트렌드 라인 차트.
Claude
[파일을 생성하고, 차트 라이브러리를 적용하고, 반응형 레이아웃을 만듭니다]

대시보드가 완성되었습니다. 브라우저에서 확인해보세요. 수정할 부분이 있으면 말씀해주세요.
나
좋은데, KPI 카드에 전주 대비 증감률도 보여줘. 그리고 다크모드도 추가해줘.

시나리오 3: 유저 리서치 데이터 분석

나
이 10개 유저 인터뷰 파일에서 공통 Pain Point 찾아줘. 빈도순으로 정리하고, 심각도도 평가해줘.
Claude
[10개 파일을 모두 읽고 분석합니다]

총 7개 Pain Point를 발견했습니다:
1. 온보딩 복잡성 (10건 중 8건 언급, 심각도: 높음)
2. 검색 기능 부재 (6건, 심각도: 높음)
3. 모바일 UX (5건, 심각도: 중간)...

각 항목별 대표 인용문과 함께 리포트를 생성했습니다.

시나리오 4: 경쟁사 분석

나
경쟁사 A, B, C의 최근 기능 업데이트를 조사해서 비교 매트릭스 만들어줘. 우리 제품과의 Gap도 분석해줘.

시나리오 5: 스프린트 리뷰 & 릴리스 노트

나
이번 스프린트 완료 티켓들을 정리하고, 외부 공개용 릴리스 노트 초안 만들어줘. 고객이 이해할 수 있는 언어로.

시나리오 6: 데이터 시각화 & 리포팅

나
이 매출 CSV 분석해서 월별 추이, 제품군별 비중, YoY 성장률 차트로 만들어줘. 경영진 보고용.

6. 한 단계 더: 진짜 파워가 나오는 기능들

여기서부터가 ChatGPT와 완전히 차원이 달라지는 부분입니다. "한 번 물어보고 답 받기"를 넘어서, 팀 전체의 업무 방식을 바꿀 수 있는 시스템 기능들입니다.

6-1. CLAUDE.md — "우리 팀 규칙을 AI에게 학습시키기"

ChatGPT와의 차이 ChatGPT는 매번 새 대화를 시작할 때마다 맥락을 처음부터 설명해야 합니다. Claude Code는 프로젝트 폴더에 CLAUDE.md 파일을 두면 매번 자동으로 읽습니다. 한 번 써두면 팀 전체가 같은 맥락으로 AI를 사용할 수 있습니다.

CLAUDE.md는 프로젝트 루트에 두는 마크다운 파일입니다. Claude가 세션을 시작할 때 자동으로 이 파일을 읽고, 그 규칙에 맞춰 작동합니다.

# CLAUDE.md 예시 (PM 팀용)

## 프로젝트 개요
- 서비스명: OO커머스
- B2C 이커머스 플랫폼
- 월 활성 사용자 50만명

## 기술 스택
- Frontend: Next.js + TypeScript
- Backend: Supabase
- 배포: Vercel

## 문서 작성 규칙
- PRD는 Lenny's Template 형식으로
- 사용자 스토리는 "As a [역할], I want [기능], so that [가치]" 형식
- 우선순위는 RICE 프레임워크 사용
- 한국어로 작성, 기술 용어는 영어 병기

## 우리 팀 컨텍스트
- 현재 스프린트: Sprint 24 (3/17~3/28)
- 주요 목표: 결제 플로우 개선
- 제약사항: PG사 연동은 KG이니시스

이렇게 한 번 만들어두면, "PRD 만들어줘"라고만 해도 Claude가 우리 서비스 맥락, 문서 형식, 팀 컨벤션을 이미 알고 작업합니다. 매번 배경 설명을 할 필요가 없습니다.

PM 팀에서 CLAUDE.md에 넣으면 좋은 것들
  • 서비스/제품 개요, 타겟 사용자
  • PRD/스펙 문서 템플릿과 형식
  • 우선순위 프레임워크 (RICE, MoSCoW 등)
  • 현재 스프린트 목표와 주요 이슈
  • 자주 쓰는 용어 정의 (사내 용어 사전)
  • 경쟁사 목록과 포지셔닝

6-2. Skills & Custom Commands — "반복 업무를 원클릭으로"

매주 스프린트 리뷰 때마다 같은 형식으로 정리하나요? 매번 같은 포맷으로 릴리스 노트 쓰나요? Skills와 Custom Commands를 만들면 이런 반복 업무를 한 번의 명령으로 실행할 수 있습니다.

예시: /sprint-review 커맨드

한 번 만들어두면 매주 재사용

/sprint-review 명령을 만들어두면:

  1. Jira에서 이번 스프린트 완료 티켓 자동 조회
  2. 기능 추가 / 버그 수정 / 개선으로 자동 분류
  3. 팀 내부용 요약 + 고객용 릴리스 노트 각각 생성
  4. Confluence에 자동 업로드

매주 1시간 걸리던 작업 → 2분

예시: /write-prd 커맨드

팀 표준 PRD를 일관되게

/write-prd "소셜 로그인" 명령을 실행하면:

  1. CLAUDE.md에 정의된 PRD 템플릿을 자동 로드
  2. 소크라틱 방식으로 누락된 요구사항 질문
  3. Edge case, 보안 고려사항 자동 포함
  4. RICE 스코어링까지 자동 계산

이런 커맨드는 팀원 누구나 같은 결과를 얻을 수 있습니다. 한 사람이 잘 만들어두면 팀 전체의 생산성이 올라갑니다.

실전 가이드: 스킬은 어디서 구하고 어떻게 만드나?

📥 이미 만들어진 스킬 가져다 쓰기

  • Claude Code에는 내장 스킬이 이미 있습니다: /simplify(코드 정리), /batch(대규모 일괄 변경), /debug(디버깅) 등
  • GitHub에서 awesome-claude-skills 같은 저장소에 100개 이상의 커뮤니티 스킬이 공유되고 있습니다
  • 팀원이 만든 스킬은 프로젝트 .claude/skills/ 폴더에 커밋하면 팀 전체가 자동으로 공유

🛠️ 직접 만드는 법 (코딩 불필요)

스킬의 실체는 SKILL.md라는 마크다운 파일 하나입니다. 핵심 구조:

── .claude/skills/sprint-review/ ──

---
name: sprint-review
description: 스프린트 리뷰 자료를 자동 생성합니다
---

이번 스프린트 완료 티켓을 분류하고 릴리스 노트를 만들어줘:
1. Jira에서 완료 티켓 조회
2. 기능추가 / 버그수정 / 개선으로 분류
3. 팀 내부용 + 고객용 릴리스 노트 각각 작성

이 파일을 만들면 /sprint-review 명령으로 바로 실행됩니다. 프로그래밍 지식이 전혀 필요 없습니다.

🤖 Skill Creator로 더 쉽게

Claude Code에 내장된 Skill Creator를 쓰면 파일도 직접 만들 필요 없습니다:

  1. "스프린트 리뷰 스킬 만들어줘"라고 말하면 Skill Creator가 SKILL.md를 자동 생성
  2. 어떤 입력을 받고, 어떤 단계로 처리할지 대화하면서 설계
  3. 성능이 마음에 안 들면? "이 스킬 개선해줘" → 자동 평가(eval) & 최적화
스킬 저장 위치에 따른 범위

.claude/skills/ (프로젝트)이 프로젝트만팀 공유 스킬 (Git에 커밋)

6-3. MCP — "AI가 우리 업무 도구에 직접 접속"

MCP(Model Context Protocol)는 Claude Code를 Jira, Slack, Google Drive 같은 외부 서비스에 직접 연결하는 기능입니다. ChatGPT에서 "Jira 티켓 정리해줘"라고 하면 "직접 접근할 수 없어요"라고 답하지만, Claude Code + MCP는 진짜로 Jira에 접속해서 데이터를 가져옵니다.

연동 도구Claude가 직접 할 수 있는 것PM 활용 시나리오
Jira / Linear 티켓 조회, 생성, 상태 변경, 코멘트 스프린트 리뷰 자동 정리, 백로그 우선순위 분석
Slack 메시지 읽기, 보내기, 채널 모니터링 버그 리포트 자동 수집, 팀 공지 발송, 일일 요약
Google Drive / Docs 문서 읽기, 편집, 생성 스펙 문서 자동 업데이트, 회의록 정리
GitHub / GitLab PR 리뷰, 이슈 관리, 코드 분석 릴리스 노트 자동 생성, PR 상태 추적
Notion 페이지 읽기, 생성, 수정 위키 자동 업데이트, 팀 대시보드 관리
Figma 디자인 데이터 접근 디자인 스펙 자동 추출, 디자인-개발 간극 분석
DB (PostgreSQL 등) 안전한 읽기 전용 쿼리 "이번 달 결제 성공률 추이" 자연어로 쿼리
PM
Jira에서 이번 스프린트(Sprint 24) 완료 티켓을 가져와서, 기능별로 분류하고, 고객 공지용 릴리스 노트를 만들어줘. 완성되면 Slack #product-updates 채널에 올려줘.
Claude (MCP 연동)
[Jira 접속 → Sprint 24 완료 티켓 15개 조회 → 분류 → 릴리스 노트 작성 → Slack 메시지 발송]

완료했습니다. 릴리스 노트를 #product-updates에 포스팅했습니다. 총 15개 티켓: 신기능 4개, 버그 수정 7개, 성능 개선 4개.

6-4. Agent Teams — "여러 AI가 동시에 일하기"

큰 작업을 줄 때, Claude가 혼자 다 하는 게 아니라 여러 에이전트에 업무를 분배할 수 있습니다. 리드 에이전트가 전체를 조율하고, 각 에이전트가 병렬로 작업한 뒤 결과를 합칩니다.

예시

"다음 분기 로드맵 초안 만들어줘"

리드 에이전트가 작업을 분배:

  • 에이전트 A: Jira 백로그에서 미처리 이슈 분석
  • 에이전트 B: Slack에서 최근 고객 피드백 수집
  • 에이전트 C: 경쟁사 최근 릴리스 조사
  • 에이전트 D: 현재 기술 부채 분석

→ 리드 에이전트가 결과를 종합하여 우선순위가 매겨진 로드맵 초안 완성

실전 가이드: Agent Teams 어떻게 쓰나?

🚀 시작하는 법

  1. 설정에서 CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS을 활성화 (아직 실험 기능)
  2. Claude에게 자연어로 팀 구성을 요청하면 끝
"PR #142를 리뷰할 에이전트 팀 만들어줘.
 한 명은 보안, 한 명은 성능, 한 명은 테스트 커버리지 담당으로."

Claude가 알아서 팀을 구성하고, 각 에이전트에게 역할을 배분하고, 결과를 종합합니다.

💡 서브에이전트 vs Agent Teams

서브에이전트Agent Teams
구조메인 에이전트 안에서 실행독립적인 Claude 인스턴스들
소통결과만 메인에 보고에이전트끼리 직접 대화 가능
적합한 상황단순 조사, 결과만 필요할 때복잡한 토론/협업이 필요할 때
비용낮음높음 (각각 별도 컨텍스트)

PM 입장에서: 단순 조사는 서브에이전트, 여러 관점이 필요한 분석은 Agent Teams가 적합합니다.

6-5. 스케줄 & 자동화 — "설정해두면 알아서 돌아감"

반복 작업을 스케줄로 걸어두면, 매일/매주 자동으로 실행됩니다.

자동화 예시주기하는 일
일일 버그 리포트매일 오전 9시새로 등록된 버그 티켓을 분류하고 Slack에 요약 발송
주간 스프린트 요약매주 금요일 오후 5시이번 주 완료/진행중/블로커 정리
PR 자동 리뷰PR이 올라올 때마다코드 변경사항 분석, 잠재 이슈 코멘트
월간 경쟁사 트래킹매월 1일경쟁사 앱스토어 업데이트 모니터링 & 비교

6-6. Hooks — "특정 동작에 자동 반응"

Hooks는 Claude가 특정 액션을 할 때 자동으로 실행되는 스크립트입니다. 예를 들어:

  • Claude가 파일을 수정할 때마다 → 자동으로 코드 포매팅 적용
  • 커밋하기 전에 → 자동으로 린트 체크
  • PR 생성 시 → 자동으로 테스트 실행

PM이 직접 설정할 일은 많지 않지만, 개발팀과 함께 세팅하면 품질 관리가 자동화됩니다.

요약: ChatGPT 수준 vs Claude Code 풀활용
수준하는 것생산성
ChatGPT 수준한 번 질문 → 한 번 답변 → 복붙기본
Claude Code 기본자연어로 지시 → 파일 직접 생성/수정3~5배
Claude Code 풀활용CLAUDE.md + Skills + MCP + 스케줄10배+

6-7. 자동화의 전체 그림: Command → Agent → Skill

위에서 소개한 기능들이 어떻게 엮여서 돌아가는지 전체 흐름을 보면 이해가 더 쉽습니다.

예시: 매주 금요일 스프린트 리뷰 자동화
  1. 스케줄이 매주 금요일 오후 4시에 자동 실행
  2. 커맨드 /sprint-review가 호출됨
  3. 커맨드가 서브에이전트 2개를 병렬 실행:
    • 에이전트 A: MCP로 Jira 접속 → 완료 티켓 수집 & 분류
    • 에이전트 B: MCP로 Slack 접속 → 이번 주 고객 피드백 수집
  4. 스킬이 결과를 합쳐서 릴리스 노트 생성 (CLAUDE.md에 정의된 팀 템플릿 적용)
  5. Hook이 완성된 문서를 Confluence에 자동 업로드 & Slack에 알림

이 전체 흐름이 사람 개입 없이 자동으로 돌아갑니다.

6-8. 현업에서 검증된 고급 패턴

GitHub에서 수만 개의 스타를 받은 실전 프로젝트들에서 추출한 패턴입니다.

패턴 1: "PRD 대신 프로토타입 20개"

Claude Code 창시자 Boris Cherny의 조언

스펙 문서를 정교하게 쓰는 대신, Claude Code로 프로토타입을 20~30개 빠르게 만들어서 비교하라. 비용이 낮으니 많은 시도를 하는 게 더 효과적이다.

실제로 Boris의 팀은 하루 141개 PR을 처리하며, PR 하나의 중간값은 118줄 — 작고 빠른 반복을 극단적으로 실천하고 있습니다.

패턴 2: Chief-of-Staff 에이전트

PM을 위한 커뮤니케이션 자동화

매일 쏟아지는 Slack, 이메일, Jira 알림을 Chief-of-Staff 에이전트가 대신 분류:

  • 🔴 즉시 대응: P0 장애, CEO 직접 요청
  • 🟡 오늘 중: PR 리뷰 요청, 일정 확인
  • 🟢 나중에: FYI 공유, 뉴스레터

MCP로 Slack + Jira + 이메일을 연결하고, 스케줄로 매일 아침 자동 실행하면 PM의 커뮤니케이션 오버헤드가 크게 줄어듭니다.

패턴 3: 다관점 피드백 에이전트

PRD를 여러 시각에서 자동 검토

PRD 초안을 쓴 뒤, 서브에이전트를 활용해 여러 페르소나가 동시에 리뷰:

  • 엔지니어 관점: 기술적 실현 가능성, 예상 공수
  • UX 관점: 사용자 경험, 플로우 이슈
  • 경영진 관점: 비즈니스 임팩트, ROI
  • QA 관점: 테스트 시나리오, Edge case

한 명의 PM이 4가지 관점의 피드백을 동시에 받을 수 있습니다.

학습 로드맵: 어디서부터 시작할까?
단계배울 것예상 시간
Level 1슬래시 커맨드 + CLAUDE.md 메모리 + 체크포인트2시간
Level 2스킬 만들기 + Hooks + MCP 연동3시간
Level 3서브에이전트 + Agent Teams + 스케줄 자동화3시간

Level 1만 해도 충분히 생산성이 올라갑니다. 나머지는 필요할 때 하나씩 익히면 됩니다.

7. 잘 쓰는 사람의 사고법

핵심 원칙: Claude는 "똑똑한 신입사원"이다

능력은 뛰어나지만 우리 회사 맥락을 모릅니다. 잘 쓰려면 좋은 매니저처럼 지시하세요.

원칙 1: 배경을 먼저 알려줘라

이렇게 하지 마세요

"로그인 만들어줘"

이렇게 하세요

"우리는 B2C SaaS 서비스야. React + TypeScript를 쓰고, 인증은 Supabase Auth를 사용해. Google과 카카오 소셜 로그인을 지원하는 로그인 페이지를 만들어줘."

원칙 2: 한 번에 하나씩, 점진적으로

이렇게 하지 마세요

"회원가입, 로그인, 프로필, 결제, 알림 다 만들어줘"

이렇게 하세요

"먼저 회원가입 플로우부터 만들자." → 확인 → "좋아, 이제 로그인을 추가하자." → 확인 → ...

원칙 3: 계획부터 세우게 해라

이렇게 하지 마세요

"바로 코드 짜줘"

이렇게 하세요

"먼저 어떻게 구현할 건지 계획을 세워줘. 내가 확인하고 진행할게."

원칙 4: 결과를 반드시 확인하라

AI는 자신감 있게 틀릴 수 있습니다. 나온 결과를 "이게 맞아?"라는 시각으로 확인하는 건 항상 사람의 역할입니다.

원칙 5: 피드백은 구체적으로

이렇게 하지 마세요

"별로야, 다시 해줘"

이렇게 하세요

"헤더 폰트가 너무 커. 24px로 줄이고, 카드 간격을 좁혀줘. 그리고 차트 색상을 브랜드 컬러(#1a73e8)로 바꿔줘."

주의사항
  • 비밀번호, API 키, 개인정보 등 민감한 정보를 AI에 입력하지 마세요
  • AI 결과물을 검증 없이 고객에게 전달하지 마세요
  • 중요한 아키텍처 결정은 반드시 엔지니어와 논의하세요
  • 사내 보안 정책을 확인하고 사용하세요
